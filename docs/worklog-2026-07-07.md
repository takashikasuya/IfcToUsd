## セッション運用メモ

- ワークフロー: 1 Issue = TDD(red確認→実装→green) → 実データ(files/ToyodaLab.ifc)
  スクリーンショット確認 → 敵対的レビューagent → 指摘修正(+回帰テスト) → コミット →
  Issueをバイリンガル要約でクローズ。
- レビューagentは毎回実バグを検出している（無限再帰、ゴースト順序、isolate祖先可視性、
  DMS空文字列、オーバーレイ重なり）。この工程は省略しないこと。
- 「データが正しいこと」と「画面に描画されること」は別に検証する
  （CLAUDE.mdの2つの前例 + 本セッションでも複数回有効だった教訓）。
